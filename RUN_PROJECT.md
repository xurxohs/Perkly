# Руководство по запуску проекта Perkly

Для полноценного запуска проекта вам потребуется открыть **два отдельных окна терминала** (для Backend и Frontend).

Все команды должны выполняться из корневой папки проекта (`/Users/shoxrux/Desktop/STARTAP/perkly`).

---

## Терминал 1: Запуск Backend (NestJS + Prisma)

Отвечает за API и работу с базой данных. В терминале выполните по очереди следующие команды:

```bash
# Перейдите в папку backend
cd backend

# Установите зависимости (если еще не установлены)
npm install

# Сгенерируйте клиент базы данных Prisma
npx prisma generate

# Синхронизируйте структуру базы данных (SQLite/PostgreSQL)
npx prisma db push

# Запустите сервер в режиме разработки
npm run start:dev
```

*Backend сервер обычно запускается на <http://localhost:3001>*

---

## Терминал 2: Запуск Frontend (Next.js)

Отвечает за пользовательский интерфейс. В новом окне терминала выполните:

```bash
# Перейдите в папку frontend
cd frontend

# Установите зависимости
npm install

# Запустите пользовательский интерфейс в режиме разработки
npm run dev
```

*Frontend будет доступен в браузере по адресу <http://localhost:3000>*

---

## Опционально: База данных (Docker)

Если в вашем проекте для базы данных используется PostgreSQL через Docker (согласно `docker-compose.yml`), то перед запуском Backend выполните в корне проекта:

```bash
docker-compose up -d
```

*(Если у вас в `backend/prisma/schema.prisma` стоит `provider = "sqlite"`, этот шаг можно пропустить, так как база создастся локально в файле `dev.db`).*

---

## Альтернатива: Запуск одной командой

Если вы хотите запустить и Backend, и Frontend одной командой в одном (текущем) терминале (в фоновом режиме frontend и активном backend):

```bash
(cd frontend && npm run dev) & (cd backend && npm run start:dev)
```

*(Чтобы остановить серверы при таком запуске, просто нажмите `Ctrl+C`)*
